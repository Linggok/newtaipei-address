## 📤 步驟 2：上傳到 GitHub

### 方法 A：使用 Git Bash（推薦）

1. **開啟 Git Bash**
   - 在專案資料夾 `C:\Users\user\Desktop\newtaipei` 中
   - 右鍵點選空白處
   - 選擇「Git Bash Here」

2. **執行以下指令**（一次一個，按 Enter）：

```bash
# 確認位置
pwd
# 應該顯示：/c/Users/user/Desktop/newtaipei

# 加入所有檔案
git add .

# 提交檔案
git commit -m "Initial commit"

# 設定主分支
git branch -M main

# 連接 GitHub（如果出現錯誤說已存在，跳過這行）
git remote add origin https://github.com/Linggok/newtaipei-address.git

# 如果上面出現錯誤，執行這行：
# git remote set-url origin https://github.com/Linggok/newtaipei-address.git

# 上傳到 GitHub
git push -u origin main
```

3. **如果要求輸入密碼**
   - Username: `Linggok`
   - Password: **貼上 Personal Access Token**（不是密碼）
   - 如果還沒有 Token，看下面的「取得 Personal Access Token」

---

### 方法 B：使用批次檔（更簡單）

1. **雙擊執行**：`執行Git指令.bat`

## 🔑 步驟 3：取得 Personal Access Token（如果需要）

如果 `git push` 要求輸入密碼，需要 Token：

1. **前往**：https://github.com/settings/tokens
2. **點選**：「Generate new token (classic)」
3. **填寫**：
   - Note: `Render Deployment`（隨便填）
   - Expiration: `90 days`（或您想要的期限）
   - **勾選**：`repo`（全部勾選也可以）
4. **點選**：「Generate token」
5. **複製 Token**（只會顯示一次！）
6. **在 Git Bash 的密碼欄位貼上 Token**

## 🚀 步驟 5：部署到 Render

### 5.1 建立 Render 帳號

1. **前往**：https://render.com
2. **點選**：「Get Started for Free」
3. **選擇**：「Continue with GitHub」
4. **授權** Render 存取您的 GitHub

### 5.2 建立 Web Service

1. **點選**右上角「New +」
2. **選擇**：「Web Service」
3. **選擇專案**：
   - 在「Connect a repository」中
   - 選擇 `Linggok/newtaipei-address`
   - 如果沒看到，點選「Configure account」授權更多權限

### 5.3 設定部署選項

填寫以下設定：

- **Name**: `newtaipei-address`（或您喜歡的名稱）
- **Region**: 選擇離您最近的區域（例如：Singapore）
- **Branch**: `main`（預設）
- **Root Directory**: 留空（使用根目錄）
- **Environment**: `Node`
- **Build Command**: `npm install`
- **Start Command**: `npm start`
- **Plan**: 選擇 **Free**

### 5.4 開始部署

1. **確認所有設定正確**
2. **點選**：「Create Web Service」
3. **等待部署**（約 2-5 分鐘）

### 5.5 取得網址

部署完成後，您會看到：
- **URL**: `https://newtaipei-address.onrender.com`（或類似的網址）

**這就是您的手機可以使用的網址！**

## 🔧 疑難排解

### 問題：Git push 失敗

**可能原因**：
1. 需要 Personal Access Token
2. GitHub 專案不存在
3. 沒有權限

**解決方法**：
- 確認已在 GitHub 建立專案
- 使用 Personal Access Token（不是密碼）
- 確認專案名稱正確

### 問題：Render 部署失敗

**可能原因**：
1. Build Command 或 Start Command 錯誤
2. Node.js 版本不符
3. 缺少依賴

**解決方法**：
- 確認 `package.json` 中的 `start` 腳本是 `node server.js`
- 確認 Node.js 版本 >= 18
- 查看 Render 的 Logs 分頁看錯誤訊息

### 問題：網站無法開啟

**可能原因**：
1. 部署還在進行中
2. Render 免費方案休眠（15 分鐘無活動）

**解決方法**：
- 等待部署完成（狀態為 Live）
- 首次請求會較慢（約 30-60 秒喚醒），之後正常

# 部署到雲端 - 不需要電腦開著就能使用

## 問題說明

**目前的設定**：需要電腦開著並運行 `npm start`，手機才能連線使用。

**解決方案**：將應用程式部署到雲端服務，這樣：
- ✅ 不需要電腦開著
- ✅ 手機隨時可以存取
- ✅ 可以分享給其他人使用
- ✅ 有固定的網址

---

## 方案一：Vercel（最簡單，推薦）⭐

### 優點
- ✅ 完全免費
- ✅ 設定簡單
- ✅ 自動部署
- ✅ 全球 CDN（速度快）

### 步驟

#### 1. 建立 GitHub 帳號（如果還沒有）

前往：https://github.com

#### 2. 將專案上傳到 GitHub

```bash
# 在專案資料夾執行
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的帳號/newtaipei-address.git
git push -u origin main
```

#### 3. 部署到 Vercel

1. 前往：https://vercel.com
2. 使用 GitHub 帳號登入
3. 點選「New Project」
4. 選擇您的 GitHub 專案
5. 設定：
   - Framework Preset: **Other**
   - Build Command: 留空（或 `npm install`）
   - Output Directory: `public`（**重要！**）
   - Install Command: `npm install`
6. 點選「Deploy」

#### 4. 完成！

Vercel 會給您一個網址，例如：`https://newtaipei-address.vercel.app`

**⚠️ 注意**：Vercel 主要用於靜態網站，如果您的 server.js 需要後端 API，請看方案二。

---

## 方案二：Railway（支援 Node.js 後端）⭐

### 優點
- ✅ 免費額度（每月 $5 免費額度）
- ✅ 支援 Node.js 後端
- ✅ 自動部署
- ✅ 簡單易用

### 步驟

#### 1. 建立 Railway 帳號

前往：https://railway.app
使用 GitHub 帳號登入

#### 2. 建立新專案

1. 點選「New Project」
2. 選擇「Deploy from GitHub repo」
3. 選擇您的專案

#### 3. 設定環境變數（如果需要）

在 Railway 專案設定中，可以設定：
- `PORT`（通常自動設定）
- `NTPC_DATASET_OID`（如果需要）

#### 4. 部署

Railway 會自動：
- 偵測 Node.js 專案
- 執行 `npm install`
- 執行 `npm start`
- 提供公開網址

#### 5. 完成！

Railway 會給您一個網址，例如：`https://newtaipei-address.up.railway.app`

## 方案三：Render（免費，支援後端）

### 優點
- ✅ 免費方案可用
- ✅ 支援 Node.js 後端
- ✅ 自動部署

### 步驟

#### 1. 建立 Render 帳號

前往：https://render.com
使用 GitHub 帳號登入

#### 2. 建立新 Web Service

1. 點選「New +」
2. 選擇「Web Service」
3. 連接 GitHub 專案
4. 設定：
   - Name: `newtaipei-address`
   - Environment: `Node`
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Plan: **Free**

#### 3. 部署

Render 會自動部署並提供網址

## 方案四：Heroku（傳統選擇）

3. 執行：

